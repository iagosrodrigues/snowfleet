# Secrets with agenix-rekey

This repository now uses `agenix` + `agenix-rekey` for secret management.

## Layout

- `secrets/*.age`: source secrets encrypted with your master key (YubiKey)
- `secrets/rekeyed/<hostname>/`: host-specific encrypted secrets generated by `agenix rekey`

## One-time setup (master identity on YubiKey)

1. Generate a split identity with `age-plugin-yubikey`.
2. Store the identity file at:
   - `/persist/secrets/agenix/master-iago-yubikey.txt`
3. Keep this file private.

Reference commands:

```bash
nix shell nixpkgs#age-plugin-yubikey
age-plugin-yubikey --generate -o /persist/secrets/agenix/master-iago-yubikey.txt
age-plugin-yubikey --identity /persist/secrets/agenix/master-iago-yubikey.txt
```

## Daily workflow

```bash
# Open dev shell with agenix wrapper
nix develop

# Create/edit source secret encrypted to your YubiKey identity
agenix edit secrets/example.age

# Rekey all used secrets for hosts and add generated files
agenix rekey -a
```

## iago password secret

The host now reads `users.users.iago.hashedPasswordFile` from agenix.

1. Generate a hash (example):

```bash
mkpasswd -m sha-512
```

2. Replace the repository secret:

```bash
printf '%s' '$6$...your-hash...' | agenix edit -i /dev/stdin secrets/iago-password.age
agenix rekey -a
```

3. Rebuild:

```bash
sudo nixos-rebuild switch --flake .#hellplace
```

## Wiring a secret in NixOS

In a module:

```nix
{
  age.secrets.example.rekeyFile = ../../secrets/example.age;
}
```

Then consume it via `config.age.secrets.example.path`.
